# Что нужно, чтобы выложить сайт на GitHub (Pages) за вас

Я **не могу сам** зайти в ваш GitHub без доступа. Нужно одно из ниже.

## Вариант 1 — вы даёте доступ (удобно один раз)

1. **Создайте репозиторий** на GitHub (пустой, без README, если будете пушить существующий проект).
2. **Personal Access Token (PAT)** с правами `repo` (классический токен) или через **Fine-grained token** с доступом к этому репозиторию.
3. **Сообщите мне** (в личном сообщении, не в публичном чате):
   - URL репозитория, например `https://github.com/ВАШ_ЛОГИН/ИМЯ_РЕПО`
   - либо дайте возможность через **GitHub Codespace / совместный доступ**, если не хотите передавать токен.

С токеном можно один раз выполнить `git push` из вашей среды (или вы сами выполните команды из `README.md`).

## Вариант 2 — вы пушите сами (безопаснее)

Выполните у себя на машине команды из раздела **GitHub + GitHub Pages** в `README.md` (создать репозиторий → `git remote add` → `git push`).

После push: **Settings → Pages → Source: GitHub Actions**.

## Вариант 3 — GitHub CLI (`gh`)

Если установлен `gh` и выполнен `gh auth login`, достаточно:

```bash
gh repo create ИМЯ_РЕПО --public --source=. --remote=origin --push
```

---

## Что уже настроено в проекте

- `.github/workflows/deploy-pages.yml` — сборка и публикация на **GitHub Pages** при push в `main` / `master`.
- В репозиторий **не** попадает `node_modules` — на сервере GitHub ставятся зависимости через `npm ci` по `package-lock.json`.

---

## Минимум от вас для «сам залил»

| Что | Зачем |
|-----|--------|
| Логин GitHub и **имя репозитория** | куда пушить |
| **PAT** или вы сами делаете `git push` | авторизация |
| Включить **Pages → GitHub Actions** в настройках репо | чтобы сайт открылся по ссылке `https://ВАШ_ЛОГИН.github.io/ИМЯ_РЕПО/` |

**Токены никогда не публикуйте** в открытом коде или в чатах — только в защищённых секретах или локально.
